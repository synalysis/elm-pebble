# Core Oracle Harness

This harness is a staging step for `m3_5-elm-test-oracle`.

It runs:

1. The official Elm `elm/core` test suite from `core-1.0.5/tests` via `elm-test` (when available locally).
2. The `elmc` core conformance suite in ExUnit (`core_compliance_test` + `core_differential_conformance_test`).

## Run

From repository root:

```bash
bash "elmc/scripts/run_core_oracle.sh"
```

Optional:

```bash
bash "elmc/scripts/run_core_oracle.sh" --skip-elm
bash "elmc/scripts/run_core_oracle.sh" --skip-elmc
```

If `core-1.0.5/tests` is not present, the script automatically skips the official
`elm-test` run and still executes the `elmc` suite.

## Artifacts

Output files are written to:

- `elmc/tmp/core-oracle/elm-test.json`
- `elmc/tmp/core-oracle/elmc-core.log`
- `elmc/tmp/core-oracle/report.md`

`report.md` is generated by `elmc/scripts/compare_core_oracle.py` and summarizes:

- official `elm-test` totals (when report exists and is parseable),
- `elmc` core-suite totals,
- parsed test entries from both sides,
- parsing/run notes for missing or malformed artifacts.

## Current scope

This is an oracle **harness bootstrap**, not yet a test-by-test differential matcher.
It provides a single command that executes both reference and `elmc` suites with stable artifact locations.
