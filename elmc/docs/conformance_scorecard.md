# Conformance Scorecard (M7)

The conformance scorecard is generated by:

```bash
mix test test/conformance_scorecard_test.exs
```

This test does two things:

1. Builds a fresh `elmc` output for the `simple_project` fixture.
2. Computes and writes scorecard artifacts:
   - `elmc/test/tmp/conformance/scorecard.json`
   - `elmc/test/tmp/conformance/scorecard.md`

It also enforces guardrails to prevent regression:

- minimum required generated function symbols,
- minimum required runtime symbol inventory,
- minimum behavior assertion count in the compliance harness,
- and full presence of required symbols in generated/runtime output.

If any guardrail drops below baseline, the test fails.

## Release Guardrail Command

For a release-style check (generate scorecard + compare against stored baseline), run:

```bash
bash "elmc/scripts/run_conformance_guardrails.sh"
```

This command:

1. Runs `test/conformance_scorecard_test.exs`.
2. Compares current metrics with `elmc/docs/conformance_baseline.json`.
3. Writes a delta report to `elmc/test/tmp/conformance/guardrail_report.md`.
4. Exits non-zero on regression.
